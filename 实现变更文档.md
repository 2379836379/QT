# oj-client 实现变更文档

本文档记录本轮在 `oj-client` 上完成的所有功能与工程化改动，分为两批：

1. **低风险增量功能**（题目页增强、收藏导入导出、可配置服务地址）
2. **本地刷题管理 + 工程化**（标签/笔记、任务看板、统计/错题本、测试/CI/打包）

所有改动均为**纯增量**，不修改提交、判题、AI、收藏等既有核心逻辑。

## 二、本地刷题管理（标签/笔记 + 任务看板 + 统计/错题本）

### 架构

```
ProblemPage 笔记侧栏 ──┐
TaskBoardPage 任务看板 ─┼── ProblemMetaService ── ProblemMetaRepository ── data/problemmeta.db
StatsPage 统计/错题本 ──┘
         ↑
    MainWindow（setCurrentProblem 时载入/保存）
```

### 2.1 阶段 1：共享本地元数据层

#### 新增文件

| 文件 | 说明 |
|------|------|
| `src/repository/meta/problemmetarepository.h` | 数据结构 `ProblemMeta`、仓库接口声明 |
| `src/repository/meta/problemmetarepository.cpp` | SQLite 读写实现 |
| `src/service/meta/problemmetaservice.h` | 服务层接口 |
| `src/service/meta/problemmetaservice.cpp` | 薄封装，转发仓库并 `emit failed(msg)` |

#### 数据库

- **路径**：`{projectRoot}/data/problemmeta.db`
- **连接方式**：UUID 连接名（与 `FavoriteProblemRepository` 相同模式）

**表结构**：

```sql
-- 每题本地元数据
CREATE TABLE problem_meta (
    problem_url TEXT PRIMARY KEY,
    title TEXT,
    note TEXT,
    difficulty INTEGER DEFAULT 0,       -- 0=未设置, 1-5=难度
    task_status TEXT DEFAULT 'todo',    -- todo/doing/done/redo
    priority INTEGER DEFAULT 0,
    deadline TEXT,
    review_flag INTEGER DEFAULT 0,      -- 1=加入错题本
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);

-- 标签字典
CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE
);

-- 题目标签关联
CREATE TABLE problem_tags (
    problem_url TEXT NOT NULL,
    tag_id INTEGER NOT NULL,
    PRIMARY KEY(problem_url, tag_id)
);
```

#### 数据结构

```cpp
struct ProblemMeta {
    QString problemUrl;
    QString title;
    QString note;
    int difficulty = 0;
    QString taskStatus = "todo";   // 存储英文枚举
    int priority = 0;
    QString deadline;
    bool reviewFlag = false;
    QStringList tags;
};
```

#### 任务状态枚举

| 存储值 | UI 中文 |
|--------|---------|
| `todo` | 未开始 |
| `doing` | 进行中 |
| `done` | 已完成 |
| `redo` | 待重做 |

#### 仓库 API

| 方法 | 说明 |
|------|------|
| `initialize()` | 打开数据库并建表 |
| `upsertMeta(meta)` | 插入或更新题目元数据 |
| `loadMeta(url, meta*)` | 按 URL 加载（含标签） |
| `loadAllMeta()` | 加载全部，按 `updated_at` 降序 |
| `setTags(url, tagNames)` | 替换题目标签（先删后插） |
| `loadAllTags()` | 全部标签名列表 |
| `statusCounts()` | 各 `task_status` 数量 |
| `tagCounts()` | 各标签关联题数（降序） |
| `reviewProblems()` | `review_flag=1` 的题目列表 |
| `notesCount()` | 有非空笔记的题目数 |

#### MainWindow 接入

- 新增成员：`m_problemMetaRepository`、`m_problemMetaService`
- 延迟初始化块中调用 `m_problemMetaService->initialize()`
- 析构时 `delete m_problemMetaRepository`

## 五、使用指南

### 5.1 为题目添加笔记和标签

1. 打开任意题目
2. 点击左侧工具栏 **Notes** 按钮
3. 填写任务状态、难度、标签（逗号分隔）、笔记
4. 勾选「加入错题本」可将题目纳入复习列表
5. 点击 **Save** 保存到本地 `problemmeta.db`

### 5.2 使用任务看板

1. 首页左侧工具栏点击 **Task Board**
2. 四栏按任务状态分组显示所有已记录题目
3. 单击题目 → 打开题目页
4. 右键题目 → 选择「移动到: xxx」切换状态

### 5.3 查看统计与错题本

1. 首页左侧工具栏点击 **Stats**
2. 左侧查看任务状态分布、标签 Top 8、笔记数、收藏数
3. 右侧错题本列表展示所有勾选「加入错题本」的题目
4. 单击错题 → 打开题目页

### 5.4 导出 / 导入收藏

1. 进入收藏页
2. 点击 **Export** 保存为 JSON
3. 点击 **Import** 从 JSON 合并导入（同名文件夹合并）

### 5.5 配置服务地址

在 `appstate.toml` 根级添加：

```toml
openjudge_base_url = "http://your-oj-host"
judger_base_url = "http://your-judger-host:port"
email_verify_url = "http://your-verify-host:port"
```

留空则使用程序内置默认值。

## 六、约定与风险

| 约定 | 说明 |
|------|------|
| 纯增量 | 新表、新页面、新服务，不改提交/判题/AI/收藏既有逻辑 |
| 样式复用 | 新控件复用现有 `objectName`，自动继承深浅色 QSS |
| 错题本来源 | 本地 `review_flag`，不依赖提交记录抓取，零网络风险 |
| 任务看板 | 先做列表 + 右键切换状态，不做拖拽（降低复杂度） |
| 统计图表 | 不引入 QtCharts，用 `QFrame` 拉伸条形 |
| 编译验证 | 本机若无 Qt/CMake 工具链，需在 Windows/Qt 环境或 CI 上验证 |
